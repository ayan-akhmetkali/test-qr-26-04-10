# Review brief (v1)

Короткий документ для финального ревью тестового задания.

## 1) Что закрыто в v1

- Форма на главной странице принимает URL и отправляет Ajax-запрос.
- Сервер валидирует URL и проверяет доступность ресурса.
- При успехе создаются short-code, short URL и QR.
- По короткой ссылке выполняется редирект на исходный URL.
- На переходах пишется IP-лог и увеличивается счётчик кликов.
- Добавлена публичная страница `/stats` с поиском по short-code, IP и фильтром по минимальному количеству переходов с IP.

## 2) Что принципиально НЕ делали в v1

- Не внедряли сложные архитектурные слои “на вырост”.
- Не добавляли внешние API (по условию задачи).
- Не смешивали post-v1 идеи с базовой поставкой.

## 3) Архитектурная позиция (DRY / SOLID / KISS)

- **KISS:** оставлен минимальный набор сущностей и сценариев для полного E2E потока.
- **DRY:** используем встроенные механизмы Yii2 для валидации/роутинга/response lifecycle.
- **SOLID (умеренно):** сервисы отделяют orchestration от контроллера, без лишних абстракций.

## 4) Прозрачность по тестам

- Добавлены unit-тесты на:
  - модель формы (`ShortLinkFormTest`);
  - контракт JSON-ответа `actionCreateLink` (`SiteControllerTest`).
- Если в окружении нет `vendor/bin/codecept`, это фиксируется как limitation окружения, а не как ложный “passed”.
