## Позиционирование текущего результата

- Это **v1**: закрыты ключевые end-to-end сценарии (создание short link, QR, redirect, click tracking).
- Использованы штатные возможности Yii2 там, где они уже решают задачу (без переизобретения framework-слоя).
- Доменная логика (short code policy, availability check, redirect tracking) вынесена отдельно.
- Отложенные задачи (post-v1) зафиксированы явно в архитектурном документе, чтобы не смешивать scope.

## Acceptance checklist (v1)

- Форма на главной странице: URL + кнопка `OK`.
- Валидация URL + проверка доступности ресурса.
- Сообщение `Данный URL не доступен` при недоступности.
- Успешный сценарий: запись в БД + short link + QR.
- Ответ через Ajax без перезагрузки.
- Редирект по короткой ссылке на original URL.
- Лог IP перехода и инкремент счётчика.
- Без API сторонних серверов.
- Наличие инструкции по развёртыванию.
